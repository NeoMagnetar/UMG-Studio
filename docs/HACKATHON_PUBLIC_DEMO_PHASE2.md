# Hackathon Public Demo Phase 2

## Purpose

Phase 2 adds a public-facing UMG landing and intake shell in front of the existing internal Studio. The goal is to make UMG Studio open like a cognitive runtime product while preserving the graph editor, Debug Mode, Clear Graph, tile canvas, MOLT Blocks, compile, and generate behavior.

The public message is:

- Universal Modular Generation
- Agentic Modular Cognition
- Upload or describe a workflow, business process, chatbot, agent, or cognitive system
- UMG will generate a structured cognitive Sleeve in a later phase
- Hermes will execute through that Sleeve and return traceable runtime activation when real runtime wiring is connected

## No fake runtime rule

This phase does not fake Sleeve generation, Hermes execution, trace playback, completion, or block activation. Intake values are captured locally and the page reports that analyzer and Sleeve assembly wiring are next-phase work.

## Public vs Studio/Debug split

The app opens to a public landing/intake shell by default. The existing Studio remains accessible through explicit controls:

- Open Studio
- Enter Builder
- Debug Studio

The Studio header also includes a return path to the public intake shell.

## Runtime foundation status

The public page can show Phase 1 foundation readiness and whether a Hermes endpoint is configured, but it does not call Hermes from the public shell.
